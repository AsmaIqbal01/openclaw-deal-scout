# Implementation Plan: OpenClaw MCP Gateway + Dashboard

**Branch**: `005-mcp-dashboard` | **Date**: 2026-07-23 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-mcp-dashboard/spec.md`

---

## Summary

Expose the existing 4-step pipeline (Gmail → Gemini → HubSpot → Discord) as a FastMCP HTTP server (`openclaw-gateway`) running on port 18789. The gateway serves OpenClaw's built-in dashboard with six MCP tools covering pipeline status, deal records, quota usage, health checks, and a manual run trigger. A new `openclaw` CLI (entry-point script, no Claude Code dependency) provides `gateway status`, `dashboard`, and `doctor` subcommands. An automated grep test enforces zero Claude Code references in all production source files.

---

## Technical Context

**Language/Version**: Python 3.12 (matches existing venv)
**Primary Dependencies**:
- `fastmcp>=2.0` (already installed, v3.4.4 confirmed) — HTTP transport via `mcp.run(transport="http", host=host, port=port)`
- `portalocker>=2.8` (already installed) — shared read lock on state store
- `google-auth-oauthlib` (already installed) — Gmail OAuth check in `get_health`
- `requests>=2.31` (already installed) — HubSpot + Discord reachability checks in `get_health`
- `pipeline_orchestrator` (existing package) — `runner.run_cycle()`, `CycleLogger`, `PipelineConfig`
- stdlib only: `threading`, `argparse`, `webbrowser`, `json`, `pathlib`, `os`, `datetime`, `logging`

**New env vars**:
- `GATEWAY_HOST` (default `127.0.0.1`)
- `GATEWAY_PORT` (default `18789`)
- `SCHEDULER_MODE` extended with new `"gateway"` value

**Storage**: Same `processed_ids.json` and `pipeline.log` as existing pipeline (read-only access from gateway)

**Testing**: `pytest`, `pytest-asyncio` (asyncio_mode=auto) — same harness as existing suite

**Target Platform**: Ubuntu 22.04 (WSL2 + native Linux)

**Performance Goals**:
- `get_gateway_status` / `get_pipeline_cycles` / `get_deals` / `get_quota_usage`: respond within 500ms
- `get_health`: completes within 30 seconds (per-component timeout 10s)
- `run_cycle`: blocks until cycle completes (1–10 min on free-tier Gemini); gateway remains responsive to other tool calls during this time (tool runs in background thread)

**Constraints**:
- Zero new paid dependencies (Constitution Principle I)
- State store reads use `portalocker.LOCK_SH` (shared) — never LOCK_EX in gateway
- Gateway does not write to `processed_ids.json`; only pipeline steps do
- FastMCP HTTP server and scheduler loop share a single process via `threading.Thread`

## Architecture Decisions

All decisions recorded in full at [research.md](research.md). Summary:

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | FastMCP HTTP `transport="http"` on port 18789 | Only transport that serves a persistent HTTP endpoint; zero-cost; already in deps |
| 2 | New `src/openclaw_gateway/` package | Mirrors existing package structure; independently testable |
| 3 | Background scheduler thread + FastMCP HTTP in main thread | Avoids two processes; thread-safe with `CycleLock`; avoids asyncio event-loop nesting |
| 4 | `openclaw` entry-point script via `pyproject.toml` | Standard Python CLI pattern; zero new dependencies (argparse + webbrowser stdlib) |
| 5 | 6 MCP tools: status, run, cycles, deals, quota, health | Covers all dashboard FRs and CLI commands with minimal surface |
| 6 | Claude Code independence test (`test_claude_code_independence.py`) | Automated enforcement; runs with every `pytest` invocation |
| 7 | `GATEWAY_HOST` + `GATEWAY_PORT` env vars; port 18789 default | Matches OpenClaw built-in dashboard URL; configurable without code changes |

### Updated systemd Unit

The `deploy/openclaw.service` changes from `Type=oneshot` to `Type=simple` and invokes `openclaw_gateway` instead of `pipeline_orchestrator`. The timer (`openclaw.timer`) is retained for backward compatibility but becomes optional — the gateway handles its own scheduling loop in `gateway` mode.

```ini
[Service]
Type=simple
ExecStart=/usr/bin/python3.12 -m openclaw_gateway
Environment=SCHEDULER_MODE=gateway
```
